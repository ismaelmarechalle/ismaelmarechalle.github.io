Port Turnaround Time Analysis

Excel dashboard identifying vessel congestion patterns and operational bottlenecks from 6 months of simulated West African port call data.

📊 Deliverable: Excel dashboard + 1-page ops report
📁 Dataset: 280 simulated vessel calls
⚓ Context: West African port operations

The Problem

Port congestion is one of the top cost drivers in maritime logistics. Every hour a vessel waits at anchor or idles at berth is demurrage the shipowner pays — and delay the cargo owner absorbs in their supply chain. Port operations teams need a clear view of where time is lost: is it waiting for a berth? Slow cargo handling? Customs clearance? Documentation?

This project builds a structured analysis tool that breaks down vessel turnaround time into its component phases and identifies where the bottlenecks are concentrated.

Dataset

The dataset simulates 280 vessel calls over 6 months at a fictional West African port. Each row represents one vessel call with the following fields:

  • Vessel ID & name — anonymized identifiers
  • Vessel type — container, bulk carrier, tanker, RoRo, general cargo
  • Cargo type — dry bulk, liquid bulk, containerized, vehicles
  • ETA / ATA — estimated and actual time of arrival
  • Berth arrival / departure — when vessel moved from anchorage to berth
  • Operations start / end — cargo handling window
  • ATD — actual time of departure

The dataset is reproducible — generated with a fixed random seed so anyone can download the Excel file and verify the numbers.

What the Dashboard Shows

  1. Summary KPIs tab — average total turnaround time, average waiting time at anchor, average cargo operations time, on-berth idle time
  2. By Vessel Type tab — pivot table + bar chart comparing turnaround across vessel types; container ships vs. bulk carriers tell very different stories
  3. Monthly Trend tab — line chart showing whether performance improved or degraded over the 6-month window
  4. Peak Congestion tab — heatmap by day-of-week and hour showing when the port is most congested
  5. Bottleneck Identification tab — waterfall chart decomposing average turnaround into: waiting at anchor → berthing maneuver → cargo ops → documentation → departure clearance

1-Page Operations Report

Alongside the dashboard, the project includes a styled PDF "Port Performance Summary" formatted like an actual operations report. It contains:

  • 3-sentence executive summary
  • Top 3 bottleneck findings with supporting numbers
  • 2 recommendations for the port operations team

This demonstrates the ability to communicate analytical findings to a non-technical audience — a skill that matters as much as the analysis itself.

Excel Techniques Used

  • Pivot tables with calculated fields for time differences
  • Dynamic charts linked to slicers (vessel type, month, cargo type)
  • Conditional formatting for the congestion heatmap
  • AVERAGEIFS, COUNTIFS, TEXT functions for KPI calculations
  • Named ranges for clean formula references

Connection to Real Experience

This project is directly inspired by observations from my port operations internship. The waiting time at anchor, the documentation bottleneck, the Monday congestion spike — these are patterns I saw in real port operations reporting. The numbers are simulated; the logic is real.
